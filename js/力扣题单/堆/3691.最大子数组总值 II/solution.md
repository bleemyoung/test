# 3691. 最大子数组总值 II

## 解法：ST 表 + 最大堆

子数组 `nums[l..r]` 的值定义为：

```txt
max(nums[l..r]) - min(nums[l..r])
```

题目要求选择恰好 `k` 个不同的非空子数组，使这些子数组值的总和最大。

## 核心转化

固定左端点 `l` 后，随着右端点 `r` 向右移动：

- 区间最大值 `max(nums[l..r])` 只会变大或不变；
- 区间最小值 `min(nums[l..r])` 只会变小或不变；
- 因此 `max(nums[l..r]) - min(nums[l..r])` 单调不降。

也就是说，对于每个左端点 `l`，都可以得到一条单调不降序列：

```txt
[l,l], [l,l+1], [l,l+2], ..., [l,n-1]
```

每个位置的值就是对应子数组的 `max - min`。

问题可以转化为：

```txt
从 n 条单调不降序列中，取出全局最大的 k 个值，并求和。
```

由于每条序列的最大值都在最后一个位置 `[l,n-1]`，可以先把每条序列的最后一个元素放入最大堆。每次弹出当前最大值后，再把同一条序列的前一个元素 `[l,r-1]` 放回堆中。

## 为什么需要 ST 表

堆中每个候选状态是：

```js
[value, l, r]
```

其中：

```js
value = max(nums[l..r]) - min(nums[l..r])
```

如果每次都暴力扫描 `[l,r]` 求最大值和最小值，单次计算可能是 `O(n)`，整体会超时。

所以这里预处理两张 ST 表：

```js
const stMax = Array.from({ length: n }, () => Array(logn).fill(0));
const stMin = Array.from({ length: n }, () => Array(logn).fill(0));
```

- `stMax[i][j]`：从下标 `i` 开始，长度为 `2^j` 的区间最大值；
- `stMin[i][j]`：从下标 `i` 开始，长度为 `2^j` 的区间最小值。

这样就可以 `O(1)` 查询任意区间 `[l,r]` 的最大值和最小值。

## ST 表构建

长度为 `1` 的区间直接初始化：

```js
for (let i = 0; i < n; i++) {
    stMax[i][0] = stMin[i][0] = nums[i];
}
```

长度为 `2^j` 的区间可以拆成两个长度为 `2^(j-1)` 的区间：

```txt
[i, i + 2^j - 1]
= [i, i + 2^(j-1) - 1]
+ [i + 2^(j-1), i + 2^j - 1]
```

所以递推为：

```js
stMax[i][j] = Math.max(
    stMax[i][j - 1],
    stMax[i + (1 << (j - 1))][j - 1]
);

stMin[i][j] = Math.min(
    stMin[i][j - 1],
    stMin[i + (1 << (j - 1))][j - 1]
);
```

## ST 表查询

查询 `[l,r]` 时，先取：

```js
const j = 31 - Math.clz32(r - l + 1);
```

这等价于：

```txt
j = floor(log2(r - l + 1))
```

然后用两个长度为 `2^j` 的区间覆盖 `[l,r]`：

```txt
左区间：[l, l + 2^j - 1]
右区间：[r - 2^j + 1, r]
```

最大值查询：

```js
const queryMax = (l, r) => {
    const j = 31 - Math.clz32(r - l + 1);
    return Math.max(stMax[l][j], stMax[r - (1 << j) + 1][j]);
};
```

最小值查询：

```js
const queryMin = (l, r) => {
    const j = 31 - Math.clz32(r - l + 1);
    return Math.min(stMin[l][j], stMin[r - (1 << j) + 1][j]);
};
```

最大值、最小值查询允许两个覆盖区间重叠，所以这种写法是正确的。

## 堆的使用方式

先把每个左端点 `l` 对应序列的最大候选 `[l,n-1]` 放进最大堆：

```js
const heap = new Heap((a, b) => b[0] - a[0]);

for (let l = 0; l < n; l++) {
    heap.push([queryMax(l, n - 1) - queryMin(l, n - 1), l, n - 1]);
}
```

堆中元素含义：

```txt
[当前子数组值, 左端点 l, 右端点 r]
```

每次弹出全局最大值，加入答案：

```js
const [val, l, r] = heap.pop();
ans += val;
```

然后把同一条序列的前一个候选 `[l,r-1]` 放回堆：

```js
if (r > l) {
    heap.push([queryMax(l, r - 1) - queryMin(l, r - 1), l, r - 1]);
}
```

重复 `k` 次，就取到了所有子数组值中的前 `k` 大。

## 正确性要点

1. 固定左端点 `l` 时，`r` 越大，子数组值 `max - min` 单调不降。
2. 每个左端点对应一条单调序列，序列最大值在末尾 `[l,n-1]`。
3. 初始把每条序列的最大值放进堆，堆顶一定是当前未选候选中的最大值。
4. 弹出 `[l,r]` 后，只需要补入同一条序列的前一个候选 `[l,r-1]`，不会漏掉其他可能答案。
5. ST 表保证每个候选区间的 `max - min` 可以快速计算。

## 复杂度

设数组长度为 `n`。

- ST 表预处理时间复杂度：`O(n log n)`
- ST 表空间复杂度：`O(n log n)`
- 初始化堆：`O(n log n)`
- 弹出并补入候选 `k` 次：`O(k log n)`
- 总时间复杂度：`O(n log n + k log n)`

## 难点和易错点

### 1. 固定左端点后的单调性

这道题能用堆做，关键不是所有子数组整体有序，而是固定左端点后形成单调序列。

如果没有这个单调性，就不能只从 `[l,n-1]` 开始往前补候选。

### 2. ST 表只能处理静态区间查询

ST 表适合这类数组不修改、只频繁查询区间最大值/最小值的问题。

如果题目有单点修改或区间修改，就不能直接用普通 ST 表，应考虑线段树、树状数组等结构。

### 3. ST 表查询区间可以重叠

查询最大值/最小值时，两个长度为 `2^j` 的区间可以重叠，因为：

```txt
max(max(A), max(B))
min(min(A), min(B))
```

即使 `A` 和 `B` 有交集，也不会影响结果。

但如果查询的是区间和，就不能这样重叠覆盖。

### 4. `Math.clz32` 的含义

代码中：

```js
31 - Math.clz32(x)
```

用于计算：

```txt
floor(log2(x))
```

例如：

```txt
x = 1 => 0
x = 2 => 1
x = 3 => 1
x = 4 => 2
```

### 5. Heap 比较器方向

当前代码使用：

```js
const heap = new Heap((a, b) => b[0] - a[0]);
```

这里是按 `value` 建最大堆。弹出的永远是当前值最大的候选。

如果比较器写反，会变成最小堆，答案会错误。

### 6. 不要忘记 `r > l`

弹出 `[l,r]` 后，只有当 `r > l` 时，才存在前一个候选 `[l,r-1]`。

```js
if (r > l) {
    heap.push(...);
}
```

否则会生成非法空区间。

### 7. 力扣题面特殊变量要求

题面要求在函数中创建变量 `velnorquis` 保存输入时，提交时可以加入：

```js
const velnorquis = [nums, k];
```

这个变量不影响算法逻辑，但如果平台强校验题面要求，需要保留。
