# 3660. 跳跃游戏 IX - 官方题解 1 代码理解

这份笔记只解释你当前 `solution.js` 这版代码怎么对应题解思路，重点看这几个点：

- `prevMax` 是什么
- `reduce` 在干什么
- `process(r, rightMin, rightMax)` 三个参数分别代表什么
- 为什么这段代码没有显式写出 `ans[i] = ans[i + 1] / preMax[i]`，但本质还是同一个结论

---

## 1. 先把题解结论翻译成更容易落地的话

题解的核心结论是：

对于某条分界线：

- 左边一段是 `[0..i]`
- 右边一段是 `[i+1..n-1]`

如果满足：

```text
左边最大值 <= 右边最小值
```

那么左边任何位置都跳不到右边。

原因是：

- 想从左边跳到右边，必须一上一下交替跳
- 但如果左边所有数都不大于右边所有数，那么左边没有一个数能“压过”右边
- 所以左边这整段和右边断开了

于是：

- 左边这一段的答案，都只能在左边内部产生
- 这时这段里的最大可达值就是“左边这段的最大值”

反过来，如果：

```text
左边最大值 > 右边最小值
```

说明左右两段是连通的。

这时：

- 左边这一段能到达的最大值
- 和右边这一段能到达的最大值

本质上是同一个值。

所以题解才有这个转移：

```text
ans[i] =
  preMax[i],     preMax[i] <= sufMin[i+1]
  ans[i+1],      preMax[i] >  sufMin[i+1]
```

## 2. 这份 JS 代码不是按 `i = n-1 -> 0` 逐个写，而是“按块处理”

你的代码：

```js
var maxValue = function(nums) {
    const n = nums.length;
    const ans = new Array(n).fill(0);

    const prevMax = new Array(n);

    nums.reduce(
        (prev, value, index) => {
            if (value > prev[0]) {
                prev = [value, index];
            }
            return (prevMax[index] = [...prev]);
        },
        [-Infinity, -1],
    );

    const process = (r, rightMin, rightMax) => {
        const [pMax, pivotIndex] = prevMax[r];
        const currMax = pMax <= rightMin ? pMax : rightMax;

        let nextRightMin = Math.min(pMax, rightMin);
        for (let i = pivotIndex; i <= r; i++) {
            ans[i] = currMax;
            nextRightMin = Math.min(nextRightMin, nums[i]);
        }

        if (pivotIndex === 0) {
            return;
        }

        process(pivotIndex - 1, nextRightMin, currMax);
    };

    process(n - 1, Infinity, 0);

    return ans;
};
```

它和题解的区别只是：

- 题解写法：从右往左，一个一个位置算
- 这份代码：把一整段答案相同的位置一次性填掉

所以这份代码本质是“题解结论的分段实现版”。

## 3. `prevMax` 是什么

### 3.1 定义

`prevMax[i]` 表示：

```text
nums[0..i] 这一段中的最大值，以及这个最大值第一次出现的位置
```

代码里存的是二元组：

```js
[最大值, 最大值下标]
```

例如：

```js
nums = [6, 8, 5, 9, 7]
```

那么：

```js
prevMax = [
  [6, 0],
  [8, 1],
  [8, 1],
  [9, 3],
  [9, 3]
]
```

意思是：

- 到下标 `0` 为止，最大值是 `6`，位置在 `0`
- 到下标 `1` 为止，最大值是 `8`，位置在 `1`
- 到下标 `2` 为止，最大值还是 `8`，位置还在 `1`
- 到下标 `3` 为止，最大值变成 `9`，位置在 `3`
- 到下标 `4` 为止，最大值还是 `9`，位置在 `3`

### 3.2 为什么代码里需要“最大值下标”

因为后面要按块填答案。

如果在区间 `[0..r]` 里，最大值出现在 `pivotIndex`，那么：

```text
[pivotIndex..r]
```

这一整段会一起得到同一个答案。

所以不仅要知道最大值 `pMax`，还要知道它出现在哪。

---

## 4. `reduce` 到底在做什么

代码：

```js
nums.reduce(
    (prev, value, index) => {
        if (value > prev[0]) {
            prev = [value, index];
        }
        return (prevMax[index] = [...prev]);
    },
    [-Infinity, -1],
);
```

### 4.1 `reduce` 的三个参数

这里的：

```js
(prev, value, index)
```

分别表示：

- `prev`：上一次返回的累计结果
- `value`：当前元素值
- `index`：当前下标

初始值：

```js
[-Infinity, -1]
```

表示：

- 当前已知最大值先当作负无穷
- 当前最大值位置先记成 `-1`

### 4.2 这段逻辑逐句解释

```js
if (value > prev[0]) {
    prev = [value, index];
}
```

含义：

- 如果当前值比“到前一个位置为止的最大值”更大
- 就把当前最大值更新成 `[value, index]`

然后：

```js
return (prevMax[index] = [...prev]);
```

这一句同时做了两件事：

1. 把当前前缀最大值存到 `prevMax[index]`
2. 把它返回给下一轮，当作新的 `prev`

### 4.3 为什么要写 `[...prev]`

是为了复制一份新数组，避免引用同一个对象。

如果直接写：

```js
prevMax[index] = prev
```

虽然这题里不一定立刻出错，但容易让多个位置共享同一个数组引用，不够安全。

所以这里写成：

```js
[...prev]
```

意思就是“拍个快照”。

### 4.4 等价的普通写法

你可以把这段 `reduce` 直接理解成下面这个 `for`：

```js
let prev = [-Infinity, -1];

for (let index = 0; index < nums.length; index++) {
    const value = nums[index];
    if (value > prev[0]) {
        prev = [value, index];
    }
    prevMax[index] = [...prev];
}
```

如果你对 `reduce` 不熟，那就先按这个 `for` 理解，完全没问题。

## 5. `process(r, rightMin, rightMax)` 是整份代码最关键的地方

### 5.1 三个参数分别表示什么

```js
process(r, rightMin, rightMax)
```

表示当前正在处理区间：

```text
[0..r]
```

并且：

- `rightMin`：`[r+1..n-1]` 这一整段的最小值
- `rightMax`：`[r+1..n-1]` 这一整段最终能到达的最大值

也就是说：

- 左边是“当前还没处理的部分” `[0..r]`
- 右边是“已经处理过的部分” `[r+1..n-1]`

### 5.2 为什么初始调用是这样

```js
process(n - 1, Infinity, 0);
```

因为刚开始右边没有元素，相当于右侧区间为空：

- 空区间最小值可以视为 `Infinity`
- 空区间最大可达值这里随便给个占位值 `0`

真正第一步是否能连到右边，要看：

```js
pMax <= rightMin
```

此时 `rightMin = Infinity`，所以第一轮一定会走“断开”的逻辑，这正好符合题解里：

```text
ans[n-1] = preMax[n-1] = max(nums)
```

---

## 6. 为什么 `[pivotIndex..r]` 这一整段答案都相同

先看：

```js
const [pMax, pivotIndex] = prevMax[r];
```

它表示在区间 `[0..r]` 中：

- 最大值是 `pMax`
- 它第一次出现在 `pivotIndex`

由于 `pMax` 是 `[0..r]` 的最大值，所以在区间：

```text
[pivotIndex..r]
```

里每个位置都能到 `pivotIndex` 这个最大值位置，或者和它属于同一个连通块。

因此这一整段的答案一致，可以一次性赋值：

```js
for (let i = pivotIndex; i <= r; i++) {
    ans[i] = currMax;
}
```

这就是“按块处理”的来源。

---

## 7. `currMax` 为什么这样算

代码：

```js
const currMax = pMax <= rightMin ? pMax : rightMax;
```

完全就是题解结论的代码化版本。

### 情况 1：`pMax <= rightMin`

说明：

```text
[0..r]` 的最大值 <= 右侧的最小值
```

也就是左右断开。

那么 `[pivotIndex..r]` 这一块只能在左边内部活动，能拿到的最大值就是左边最大值：

```js
currMax = pMax;
```

### 情况 2：`pMax > rightMin`

说明左右连通。

那么当前这块和右边块能到达同样的最大值，所以：

```js
currMax = rightMax;
```

这就对应题解里的：

```text
ans[i] = ans[i + 1]
```

只不过这里不是只给某一个 `i` 赋值，而是给一整块赋值。

---

## 8. `nextRightMin` 在做什么

代码：

```js
let nextRightMin = Math.min(pMax, rightMin);
for (let i = pivotIndex; i <= r; i++) {
    ans[i] = currMax;
    nextRightMin = Math.min(nextRightMin, nums[i]);
}
```

这一段是在为下一轮递归准备新的“右侧最小值”。

当前已经处理完的是：

```text
[pivotIndex..n-1]
```

所以下一次递归会处理：

```text
[0..pivotIndex-1]
```

那么对下一轮来说，它的右边就变成了：

```text
[pivotIndex..n-1]
```

因此需要知道这整段的最小值，也就是新的 `rightMin`。

所以这里不断取最小值，最后得到：

```js
nextRightMin = min(nums[pivotIndex..n-1])
```

下一轮就传进去：

```js
process(pivotIndex - 1, nextRightMin, currMax);
```

---

## 9. 为什么下一轮的 `rightMax` 直接传 `currMax`

因为下一轮右边整段是：

```text
[pivotIndex..n-1]
```

而我们刚刚已经算出：

```text
[pivotIndex..r]
```

这一块的答案全都是 `currMax`。

右边更靠后的部分本来就已经处理过，它们所在连通块的最大可达值也已经归并进来了。

所以对下一轮左侧来说，右边整段最终能到达的最大值就是：

```js
currMax
```

因此递归传：

```js
process(pivotIndex - 1, nextRightMin, currMax);
```

## 10. 用 `nums = [6, 8, 5, 9, 7]` 手推一遍

先得到：

```js
prevMax = [
  [6, 0],
  [8, 1],
  [8, 1],
  [9, 3],
  [9, 3]
]
```

### 第一次调用

```js
process(4, Infinity, 0)
```

当前处理 `[0..4]`

```js
[pMax, pivotIndex] = prevMax[4] = [9, 3]
```

所以：

- 左边最大值是 `9`
- 最大值位置在 `3`

判断：

```js
9 <= Infinity // true
```

说明当前这块和“右边空区间”断开，所以：

```js
currMax = 9
```

于是：

```js
ans[3] = 9
ans[4] = 9
```

现在答案部分变成：

```js
ans = [0, 0, 0, 9, 9]
```

接着递归处理左边：

```js
process(2, 5, 9)
```

这里 `5` 是区间 `[3..4] = [9,7]` 的最小值。

### 第二次调用

```js
process(2, 5, 9)
```

当前处理 `[0..2]`

```js
[pMax, pivotIndex] = prevMax[2] = [8, 1]
```

判断：

```js
8 <= 5 // false
```

说明左右连通，所以：

```js
currMax = rightMax = 9
```

于是：

```js
ans[1] = 9
ans[2] = 9
```

得到：

```js
ans = [0, 9, 9, 9, 9]
```

然后递归：

```js
process(0, 5, 9)
```

### 第三次调用

```js
process(0, 5, 9)
```

当前处理 `[0..0]`

```js
[pMax, pivotIndex] = prevMax[0] = [6, 0]
```

判断：

```js
6 <= 5 // false
```

说明还能连到右边，所以：

```js
ans[0] = 9
```

最终：

```js
ans = [9, 9, 9, 9, 9]
```

这和题解结论一致。

## 11. 这份代码和题解状态转移的一一对应

题解写法：

```text
ans[i] =
  preMax[i],  preMax[i] <= sufMin[i+1]
  ans[i+1],   preMax[i] >  sufMin[i+1]
```

代码写法：

```js
const currMax = pMax <= rightMin ? pMax : rightMax;
```

对应关系：

- `pMax` 对应 `preMax[i]`
- `rightMin` 对应 `sufMin[i+1]`
- `rightMax` 对应 `ans[i+1]` 所在那一整块的答案

区别仅仅在于：

- 题解按单点转移
- 代码按整块转移

本质完全一样。

## 12. 你现在可以怎么读这份代码

推荐按这个顺序读：

### 第一步：先忽略 `reduce`

先把它脑补成：

```js
for (...) {
    ...
}
```

只理解：

```text
prevMax[i] = 前缀最大值 + 位置
```

### 第二步：只盯住 `process`

记住一句话：

```text
process(r, rightMin, rightMax) = 处理 [0..r]，已知右边那段的最小值和最终最大可达值
```

### 第三步：只看这句判断

```js
const currMax = pMax <= rightMin ? pMax : rightMax;
```

它就是整题的核心：

- 断开：答案等于左边最大值
- 连通：答案等于右边那块的答案

### 第四步：理解为什么是一整段一起赋值

```js
for (let i = pivotIndex; i <= r; i++) {
    ans[i] = currMax;
}
```

因为从 `pivotIndex` 到 `r` 属于同一块，它们答案相同。

---

## 13. 如果你只想记住最短版本

这题代码可以压缩理解成下面 4 句话：

1. `prevMax[i]` 记录 `0..i` 的最大值和位置。
2. 每次看当前左半段 `[0..r]` 的最大值 `pMax`，再和右半段最小值 `rightMin` 比较。
3. 如果 `pMax <= rightMin`，左右断开，这一块答案就是 `pMax`。
4. 如果 `pMax > rightMin`，左右连通，这一块答案就是右边那块的答案 `rightMax`。

---

## 14. 对你当前困惑的直接结论

你现在不是“题解不懂”，而是“题解的单点 DP 写法，被这份 JS 改写成了分段递归写法，所以代码不直观”。

换句话说：

- 题解结论你已经基本懂了
- 代码难点主要在“写法绕”
- 真正最需要吃透的是：
  - `prevMax`
  - `currMax = pMax <= rightMin ? pMax : rightMax`
  - `for (let i = pivotIndex; i <= r; i++)`

这三个地方一旦连起来，整份代码就通了。
