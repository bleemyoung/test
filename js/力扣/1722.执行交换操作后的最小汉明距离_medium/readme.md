# 1722. 执行交换操作后的最小汉明距离

题目链接：<https://leetcode.cn/problems/minimize-hamming-distance-after-swap-operations/description/>

题单归类：[`js/力扣题单/并查集`](../../力扣题单/并查集/review.md)

## 题意

给你两个等长数组 `source` 和 `target`，以及一些允许交换的位置对 `allowedSwaps`。

每个 `allowedSwaps[i] = [a, b]` 表示：

- `source[a]` 和 `source[b]` 可以交换
- 并且这些交换可以执行任意次、按任意顺序进行

目标是：在允许的交换范围内，尽量把 `source` 调整得和 `target` 更接近，最后求最小汉明距离。

这里的汉明距离指的是：

- 有多少个下标 `i` 满足 `source[i] !== target[i]`

---

## 一、并查集是什么

并查集，英文叫 `Union Find` 或 `Disjoint Set Union`，简称 `DSU`。

它是一个用来处理“元素分组”和“连通性判断”的数据结构，最擅长解决这两类问题：

- 两个元素是否属于同一个集合
- 把两个集合合并成一个集合

你可以把它理解成一种“朋友圈管理器”。

例如有 5 个人：

- 0
- 1
- 2
- 3
- 4

一开始每个人自己是一个组：

- `{0}`
- `{1}`
- `{2}`
- `{3}`
- `{4}`

如果已知：

- `0` 和 `1` 是朋友
- `1` 和 `2` 是朋友

那么虽然没有直接说 `0` 和 `2` 是朋友，但它们已经通过关系连起来了，所以它们属于同一个组：

- `{0, 1, 2}`

而 `3`、`4` 还是自己的组。

并查集就是专门高效维护这种“哪些点已经连成一组”的结构。

---

## 二、并查集支持什么操作

并查集核心就两个操作。

### 1. `find(x)`

查找元素 `x` 所属集合的代表节点，也叫“根节点”。

你可以把它理解成：

- 问 `x` 最终归哪个组管

如果两个元素 `x` 和 `y` 满足：

```js
find(x) === find(y)
```

就说明它们属于同一个集合。

### 2. `union(x, y)`

把 `x` 所在集合和 `y` 所在集合合并。

也就是：

- 如果这两个人本来不在一个组，就把两个组合成一个组

---

## 三、并查集内部是怎么存的

最常见的实现方式是用一个父节点数组 `fa`。

例如：

```js
fa[0] = 0
fa[1] = 0
fa[2] = 1
fa[3] = 3
```

表示：

- `0` 的父亲是自己，所以 `0` 可能是一个根
- `1` 的父亲是 `0`
- `2` 的父亲是 `1`
- `3` 的父亲是自己

那么：

- `2 -> 1 -> 0`

最终 `2` 属于以 `0` 为根的集合。

所以：

- `find(2) === 0`
- `find(1) === 0`
- `find(0) === 0`

说明 `0、1、2` 在一个集合里。

---

## 四、为什么并查集这么快

并查集常见有两个优化。

### 1. 路径压缩

查找根节点时，顺手把沿途节点直接挂到根上。

例如原来：

- `2 -> 1 -> 0`

执行一次 `find(2)` 后，可以直接变成：

- `2 -> 0`
- `1 -> 0`

这样以后再查会更快。

### 2. 按秩合并

合并两个集合时，尽量让矮的树挂到高的树下面，避免树越来越高。

在你的代码里，这个高度信息用的是 `rank`。

这两个优化加起来后，并查集在实际题目里几乎可以当成常数时间使用。

---

## 五、这道题为什么能用并查集

这题表面上是在做“交换”，但真正关键的问题不是怎么交换，而是：

- 哪些下标之间，最终可以互相到达？

例如：

```txt
allowedSwaps = [[0, 1], [1, 2]]
```

表示：

- `0` 可以和 `1` 交换
- `1` 可以和 `2` 交换

虽然没直接说 `0` 可以和 `2` 交换，但因为可以多次交换，所以：

- `0、1、2` 这三个位置实际上已经连成一个整体

这意味着什么？

这意味着这 3 个位置上的元素可以在组内任意重排。

所以这题可以转化成：

- 先用并查集找出所有“可交换连通块”
- 每个连通块内部，单独考虑它能不能把 `source` 调整得和 `target` 一致

这就是并查集在这题里的作用：**找出哪些位置属于同一个可自由交换的组。**

---

## 六、这题的核心思路

整体分 3 步。

### 第 1 步：用并查集把所有可交换位置分组

代码对应：

```js
const uf = new UnionFind(n);

for (const [a, b] of allowedSwaps) {
    uf.union(a, b);
}
```

做完后：

- `find(i)` 相同的下标，说明它们属于同一个连通块
- 同一个连通块里的位置可以通过若干次交换任意重排

### 第 2 步：统计每个组里 `source` 的元素库存

代码对应：

```js
const sets = new Map();
for (let i = 0; i < n; i++) {
    const f = uf.find(i);
    if (!sets.has(f)) {
        sets.set(f, new Map());
    }
    const cnt = sets.get(f);
    cnt.set(source[i], (cnt.get(source[i]) || 0) + 1);
}
```

这里的含义是：

- `sets` 按“连通块根节点”分组
- 每个组里再用一个 `Map` 统计 `source` 中每个数字出现了多少次

比如某个组内 `source` 的值是：

```js
[1, 2, 2, 3]
```

那么库存表就是：

```js
1 -> 1
2 -> 2
3 -> 1
```

### 第 3 步：遍历 `target`，看组内库存能不能匹配

代码对应：

```js
let ans = 0;
for (let i = 0; i < n; i++) {
    const f = uf.find(i);
    const cnt = sets.get(f);
    const count = cnt.get(target[i]) || 0;
    if (count > 0) {
        cnt.set(target[i], count - 1);
    } else {
        ans++;
    }
}
```

这段逻辑非常关键。

对于每个位置 `i`：

- 先看它属于哪个组
- 再看这个组的 `source` 库存里有没有 `target[i]` 这个值

如果有：

- 说明组内某个位置的值可以换过来
- 这个位置最终可以匹配成功
- 库存减 1

如果没有：

- 说明这个组里根本凑不出 `target[i]`
- 那这个位置无论怎么交换都匹配不了
- 汉明距离加 1

## 七、为什么“统计频次”就够了

这是这题最容易卡住的地方。

很多初学者会想：

- 不是在交换吗？
- 为什么不真的去模拟交换过程？

原因是：同一连通块内可以任意重排。

既然能任意重排，那么组内最终摆成什么样，并不重要。重要的是：

- 组里有没有足够的数字去匹配 `target` 需要的数字

所以这题本质不是“模拟交换”，而是“组内多重集合是否能匹配”。

换句话说：

- 你只需要知道这个组里有哪些值，各有几个
- 不需要知道它们原来分别站在哪个位置

这就是为什么“频次统计”足够了。

## 八、结合例子理解

例子：

```js
source = [1, 2, 3, 4]
target = [2, 1, 4, 5]
allowedSwaps = [[0, 1], [2, 3]]
```

### 1. 分组

并查集会得到两个组：

- 组 A：`{0, 1}`
- 组 B：`{2, 3}`

### 2. 统计 `source` 库存

组 A 中 `source` 的值是：

```js
[1, 2]
```

组 B 中 `source` 的值是：

```js
[3, 4]
```

### 3. 用 `target` 去匹配

`target[0] = 2`

- 位置 `0` 在组 A
- 组 A 有 `2`
- 可以匹配，库存减 1

`target[1] = 1`

- 位置 `1` 也在组 A
- 组 A 还有 `1`
- 可以匹配

`target[2] = 4`

- 位置 `2` 在组 B
- 组 B 有 `4`
- 可以匹配

`target[3] = 5`

- 位置 `3` 在组 B
- 组 B 没有 `5`
- 无法匹配

所以最终最小汉明距离是 `1`。

## 九、你的 `solution.js` 里的并查集代码解释

### 1. 初始化

```js
class UnionFind {
    constructor(n) {
        this.fa = new Array(n);
        this.rank = new Array(n).fill(0);
        for (let i = 0; i < n; i++) {
            this.fa[i] = i;
        }
    }
}
```

含义：

- `fa[i] = i` 表示一开始每个点的父亲都是自己
- 也就是一开始每个点单独成组
- `rank` 用来辅助按秩合并

### 2. 查找根节点 `find`

```js
find(x) {
    if (this.fa[x] !== x) {
        this.fa[x] = this.find(this.fa[x]);
    }
    return this.fa[x];
}
```

含义：

- 如果 `x` 的父亲不是自己，说明它还不是根
- 就继续往上找
- 找到根后，顺手把 `x` 直接挂到根上

这就是路径压缩。

### 3. 合并集合 `union`

```js
union(x, y) {
    x = this.find(x);
    y = this.find(y);
    if (x === y) return;
    if (this.rank[x] < this.rank[y]) {
        [x, y] = [y, x];
    }
    this.fa[y] = x;
    if (this.rank[x] === this.rank[y]) {
        this.rank[x]++;
    }
}
```

含义：

- 先找到 `x` 和 `y` 各自所在集合的根
- 如果根一样，说明本来就在同一组，不用合并
- 否则让秩低的树挂到秩高的树下
- 如果两个树一样高，随便挂一个，并把新的根高度加 1

## 十、时间复杂度

设：

- `n = source.length`
- `m = allowedSwaps.length`

时间复杂度近似为：

```txt
O(n + m)
```

更严格地说，并查集操作是：

```txt
O((n + m) * alpha(n))
```

其中 `alpha(n)` 是阿克曼函数的反函数，增长极慢，实际中可以近似看成常数。

空间复杂度：

```txt
O(n)
```

## 十一、初学者最容易混淆的点

### 1. 并查集管的是“位置”，不是“值”

这题合并的是数组下标：

- `0`
- `1`
- `2`

不是合并数字 `source[i]` 本身。

因为允许交换的是“位置之间可交换”。

### 2. 同组表示“可以通过若干次交换互相到达”

并不是只有直接出现在 `allowedSwaps` 里的两个点才能换。

只要通过中间点连通，也算同一组。

### 3. 不需要真的模拟交换过程

只要知道某组内有哪些值、各自有几个，就足够判断能匹配多少位置。

### 4. 匹配时要按次数扣减

例如组内只有一个 `2`，那它只能匹配一次 `target` 中的 `2`，不能重复使用。

所以代码里用了：

```js
cnt.set(target[i], count - 1);
```

---

## 十二、一句话总结

这题的并查集作用是：

- 先把所有“可以通过交换连起来的位置”分成若干组

然后利用“同组内可任意重排”这个性质：

- 统计每组中 `source` 的值出现次数
- 再去匹配这组中 `target` 需要的值
- 匹配不了的位置数，就是最小汉明距离

---

## 十三、适合初学者记住的模板理解

以后你看到下面这类描述时，就要考虑并查集：

- 哪些点属于同一组
- 哪些点彼此连通
- 给一些连接关系，问最后形成几个集合
- 两个元素是否能互相到达
- 在若干连接/合并操作后，维护连通块

这题就是非常典型的：

- `allowedSwaps` 提供边
- 下标是点
- 连通块就是可交换组

如果只记一句话，可以记成：

**并查集 = 快速维护“谁和谁在同一个集合里”的数据结构。**
